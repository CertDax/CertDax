
CertDax has been deployed to your cluster!

{{- if .Values.ingress.enabled }}

Access CertDax at:
{{- if .Values.ingress.tls.enabled }}
  https://{{ .Values.ingress.host }}
{{- else }}
  http://{{ .Values.ingress.host }}
{{- end }}

Make sure your DNS (or /etc/hosts) points {{ .Values.ingress.host }} to your
Ingress controller's external IP.

Find the external IP of your Ingress controller:

  kubectl get ingress {{ include "certdax.fullname" . }} -n {{ .Release.Namespace }}

Or check your Ingress controller's LoadBalancer service directly:

  kubectl get svc -A | grep -i ingress

Once you have the EXTERNAL-IP, create a DNS A record:

  {{ .Values.ingress.host }}  →  <EXTERNAL-IP>

Or for quick local testing, add it to /etc/hosts:

  echo "<EXTERNAL-IP> {{ .Values.ingress.host }}" | sudo tee -a /etc/hosts

{{- else }}

CertDax is not exposed externally (ingress is disabled).
To access it, use port-forward:

  kubectl port-forward svc/{{ include "certdax.fullname" . }}-frontend 8080:80 -n {{ .Release.Namespace }}

Then open http://localhost:8080 in your browser.

{{- end }}

First-time setup:
  1. Open CertDax and create your admin account
  2. Go to Settings → configure SMTP for email notifications (optional)
  3. Go to Providers → add a DNS provider or Certificate Authority
  4. Go to Certificates → request your first certificate
